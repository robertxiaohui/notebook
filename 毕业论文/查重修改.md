### 1 选题背景

#### 原版

```
2017年10月，我们迎来了党的十九大的召开，习近平总书记在会议上明确提出，要实现教育信息化，不断推进优质教育资源普及化，我们才能共同创造美好未来。其实在2003年，教育部就已经下发《教育部关于启动高等学校教学质量与教学改革工程精品课程建设工作的通知》，到了2011年10月，教育部又发布了《教育部关于建设开放教育精品课程的实施意见》，此次教育部计划在“十二五”规划期间打造1,000门精品视频公开课，还需要达到完成5,000门国家级精品资源共享课程的目标。此外国务院发布的《国家中长期教育改革和发展规划纲要(20102020年)》也提出，要促进现代教育发展，我们必须把目标瞄准在“推进现代信息技术与高等教育深度融合，创新人才培养模式”，这样才能让现代教育平稳高效发展，时间来到2015年，我国基本建立以网络资源为核心的教育公共服务体系，学习者能够享受到高效优质的网络服务教育，各种网络教学视频百花齐放，到2020年，全国大部分人民都具备了全民学习，终身学习的思想觉悟，国家也提供了全民学习、终身学习的教育体系[1]。
在2020年1月，新冠肺炎大规模爆发，这是一场突然起来的疾病，严重影响到了人们的工作学习，为了减少感染人数，预防疾病，打好这场疫情防控战役，教育部台了“停课不停学”的应急政策，学生们在家进行网络教学。此次疫情正好也推动了大规模的网络教学实践，让“互联网+教育”的技术大面积覆盖应用。顺应此次大规模的网络学习，“互联网+教育”融合已成为时代潮流[3]。

```

#### 2.2 国内研究现状

```
我国2013年发布《教育部关于开展高等学校教学质量与教学改革工程精品课程建设工作的通知》，这个通知中明确指出要进行精品课程的建设必须通过打造高质量的教学内容，建设一流的师资队伍，使用先进的教学理念的方式。教育部《高等学校教学质量与教学改革工程》离不开精品课程建设，精品课程建设作为这些工程的基石，全国各大高校都非常重视精品课程网站建设，通过精品课程网站的建设，可以更好地促进教学资源的有效利用，提高高校的教育质量和人才培养质量[4]。
2015年，教育部发布了《关于开展高校教学质量与教学改革工程精品课程建设的通知》，此次通知要求充分利用现代信息技术进行课程管理与建设，构建优质共享的教学资源。“云教育”和“智慧教育”的概念逐渐萌芽并且快速茁壮发展。近年来，精品课程建设在教学服务体系建设方面也取得了显著成效。在数据采集和统计分析方面，刘海丰等人通过各大高校的精品课程体系的仔细分析和认真研究，以 RFSTFUII、 Idea、 QT等核心技术为基础的精品课程体系逐渐完善和成熟，该体系有效地解决了精品课程体系建设中遇到的业务和技术瓶颈，输出了一套完整的精品课程管理理论体系。
经过资料的检索和关键信息的比对，如果从开发技术的角度来看，当前的精品课程网站有三种形式。
(1)	具有后端的web应用
当前，在使用开源软件基础上开发课程网站已经是越来越多的高校的选择这也是一种趋势件[9]。使用这种方式有很多的优势，因为是基于web应用的开发模式，所有学生在使用课程网站学习的时候能够有非常不错的交互体验、前后端分离的模式，容易管理和维护代码、减少服务器压力，现在大部分高效的后端共用一套程序代码，这套代码经过了检验质量能够得到保障。
(2)	单纯的静态页面设计
纯静态页面即使用html+ajax的技术，这种方式不需要在服务器上编译产生动态的网页，而是直接将html发送，在浏览器上解析，运行速度非常快，不过基于HTML只能产生静态页面，如果要长期发展，在维护方面会有极大的苦难，并且因为这种技术的静态性，有一些基本的功能比较难以实现，比如教师的答疑，学生互动交流，网上在线考试。对于安全性方面，因为是纯html页面，接口暴露无遗。
(3)	根据快速搭建平台所产生的网站
目前有一大批课程网站是基于快速搭建平台产生的，比较常见的有微助教，优课堂，精品学。这个技术最大特点是能够快速制作课程网站，开发周期很短，有专门的平台进行维护。但也存在其他的问题，因为软件是商业平台开发进行售卖的，往往开发完成之后，之后的升级换代比较慢，因为是基于快速搭建平台产生的，所以功能结构往往也比较单一，当然因为软件是商业购买的，价格也比较昂贵。目前中国大约有25%的高校是使用这种快速搭建平台开发的网站，这在一定程度上对课程教育网站的发展有制约作用。
```

### 3 研究意义

#### 3.1 理论意义

```
基于Web精品课程平台系统的开发和部署可以有效的丰富web应用软件在学校等生产办公中的应用范围，这样便可以有效的验证web应用软件的智能性[14]。另外，精品课程平台系统开发采用了JSP技术、数据库技术和 IDEA集成开发环境，能够丰富了这些计算机技术应用领域，验证了计算机技术在教学领域的应用需求。
```

#### 3.2 实践价值

##### 原版

```
当前，对现有各大高校精品课程网站进行分析，可以发现，在现有课程网站建设过程中，还存在重申报、轻建设、静态网页为主、课程资源呈现形式单一、师生互动不够等问题。怎样利用网络技术实现开放教育资源的在线化，已成为各大高校面临的时代课题。因此，本文所要实现的是一个基于 Javaweb的以教师和学生为主体，借助计算机开发技术，整合优质教学资源，降低教学成本的教学辅助系统，设计并实现基于 Javaweb的精品资源共享课程的教学辅助系统[20]。
```

### 2 关键技术简介

#### 2.1 前端技术

##### 原版

```

1.JSP(Java Server Pages)：jsp运行在服务器，jsp是服务器生成页面的一种技术，实现动态页面[5]。
JSP的运行原理：
1、先将jsp文件转译为一个类似于servlet的java类（这个转译的时间是在该页面第一次被请求时）
2、在第一请求转译之后，再被编译为class运行在tomcat容器中
3、以后所有的请求就直接运行编译好的class（也就是jsp）
所以：JSP本质上是Servlet，是Servlet的另一种形态。
JSP与Servlet的关系, JSP本质上是Servlet，两者底层代码是一致。JSP主要用于处理页面显示逻辑Servlet主要用于处理业务逻辑.两者相互合作，各自负责不同的功能，共同组成javaweb。
2.JavaScript：JavaScript简称为JS,是一种弱类型的脚本语言，由ECMAScript描述JavaScript的基本对象和语法，文档对象模型(DOM)，浏览器对象模型(BOM)三部分组成[11]。由网景公司开发的客户端脚本语言，不需要编译，可以直接运行。
Web前端分为三层：
结构层  HTML  定义页面的结构
样式层  CSS    定义页面的样式
行为层  JavaScript  用来实现交互，提升用户体验
JavaScript在web开发中的作用:在客户端动态的操作页面;在客户端做数据的校验;在客户端发送异步请求。

3.Ajax(Asynchronous Javascript And XML)：异步的JavaScript和XML，实现前后台之间的异步交互，在不需要重新加载整个页面的前提下对页面上的部分内容做局部刷新[10]。
使用JavaScript代码可以接收异步请求的响应，使用JavaScript操作响应中的数据，从而动态的修改页面。
Ajax在web开发中的作用：
1.为了改善传统的web请求(请求-->等待处理-->返回响应)
2.使用JS调用浏览器提供的一个对象(XMLHttpRequest)，该对象称之为 ajax引擎。
3.可以异步的向服务器发送请求，在异步请求处理的同时，不影响当前正在执行的请求。
4.当异步请求返回响应的时候，不需要返回整个页面，可以只返回某一个字符串或某一个对象。

```

```
MySQL数据库(Mysql5.7.26版本)：Mysql相比于sql server这样适合大型项目的数据库管理系统，在应对中小型项目的时候mysql的高效，安全，简便性体现的淋漓尽致。mysql和各大开发语言都完全适用，本论文的项目就是采用java语言开发，mysql提供了java语言的驱动，我们可以通过该驱动轻松对mysql进行操作。mysql的历史也非常悠久，于1995年诞生在瑞典的Mysql AB公司，两位开发大神David Axmark 和 Michael Monty Widenius一手建立。当然mysql的最大特点是开源，我们可以轻松的通过网络对下载mysql数据库然后进行开发， 相比于昂贵的orcale，mysql成为了更多中小型项目的首选。当然mysql也大力的发展，目前mysql的安全性和访问速度，完全能够应付大部分的项目。
Servlet规范(Servlet4.0版本): Servlet API是Java开发人员最熟悉的API之一，Servlet在1999年所发布的J2SE1.2版本中首次面世，Servlet在JavaWEB的开发中发挥着重要的作用。JavaEE8对Servlet4.0进行了重要的更新。其中服务器推送是最主要的更新，如果要使用服务器推送的功能，则我们必须使用HTTP/2.0版本的协议。JavaEE8提供了对Servlet映射的运行时发现，在运行时我们可以获取Servlet的名称，Servlet的映射路径。JavaEE8简化了对Filter的开发。


```
